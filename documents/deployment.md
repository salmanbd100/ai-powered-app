# Deployment Guide

## Stack

| Layer | Service | Free Tier |
|---|---|---|
| Frontend (React) | Vercel (static CDN) | Unlimited static deploys |

## Architecture

```
Browser → vercel.com
  /api/* → Serverless Function  (Express app — pre-bundled into api/lib/server.js)
  /*     → Static CDN           (packages/client/dist)
         ↓
    TiDB Cloud (MySQL) — add when needed
```

## Monorepo Structure

```
ai-powered-app/
├── packages/
│   ├── client/          # React frontend (its own package.json)
│   └── server/          # Express backend (its own package.json)
├── api/
│   └── index.ts         # Thin wrapper — tracked in git so Vercel detects it
│       └── lib/         # .gitignored — generated by build step
│           └── server.js
├── vercel.json
└── package.json         # Root workspace (Bun workspaces)
```

### Why This Structure

Vercel detects serverless functions from files tracked in git **before** the build runs. Server code lives in `packages/server/` (a separate workspace package), so Node.js cannot resolve cross-workspace TypeScript imports at runtime.

**Solution:** Keep `api/index.ts` in git as a thin wrapper. The build step pre-bundles all of `packages/server/` into `api/lib/server.js` using `bun build --target node`. Vercel detects the function from git, runs the build, then `@vercel/node` traces and includes the generated bundle.

```ts
// api/index.ts (tracked in git)
import app from './lib/server.js';
export default app;
```

## Deploy (Chat Feature, No Database)

### Step 1 — Push to GitHub
```sh
git add .
git commit -m "chore: initial deployment"
git push
```

### Step 2 — Import on Vercel
1. Go to [vercel.com](https://vercel.com) → **Add New Project**
2. Import your GitHub repository
3. Vercel auto-detects `vercel.json` — no manual framework config needed

### Step 3 — Set Environment Variables
In Vercel → **Settings** → **Environment Variables**, add:

| Variable | Value |
|---|---|
| `OPENAI_API_KEY` | Your OpenAI key |

### Step 4 — Deploy
Click **Deploy**. Done.

## Key Config Files

### `vercel.json`
```json
{
  "framework": null,
  "installCommand": "bun install",
  "buildCommand": "bun run build",
  "outputDirectory": "packages/client/dist",
  "rewrites": [
    { "source": "/api/(.*)", "destination": "/api/index" }
  ]
}
```

### Root `package.json` build script
```json
"build": "cd packages/client && bun install && bun run build && cd ../.. && bun build packages/server/app.ts --outfile api/lib/server.js --target node"
```

The build does two things in order:
1. Builds the React frontend into `packages/client/dist`
2. Bundles the entire Express server (all workspace dependencies included) into `api/lib/server.js`

### `.gitignore` entry
```
# vercel api bundle (generated by build step)
api/lib/
```

`api/lib/` must be gitignored — it is a build artifact. Only `api/index.ts` is tracked.

## Add Database (When Ready)

### Step 1 — Create a Free MySQL Database on TiDB Cloud
1. Sign up at [tidbcloud.com](https://tidbcloud.com)
2. Create a **Serverless** cluster (free tier)
3. Go to **Connect** → copy the connection string

Connection string format:
```
mysql://username:password@host:4000/dbname?ssl-mode=VERIFY_IDENTITY
```

### Step 2 — Add DATABASE_URL to Vercel
In Vercel → **Settings** → **Environment Variables**, add:

| Variable | Value |
|---|---|
| `DATABASE_URL` | Connection string from TiDB Cloud |

### Step 3 — Update Prisma Schema
In `packages/server/prisma/schema.prisma`, add the `url` field:

```prisma
datasource db {
  provider = "mysql"
  url      = env("DATABASE_URL")
}
```

### Step 4 — Push Schema to Database
Run once from your local machine:

```sh
cd packages/server
bunx prisma db push
```

### Step 5 — Redeploy on Vercel
Vercel will pick up the new `DATABASE_URL` on the next deploy automatically.

## Troubleshooting

| Symptom | Likely Cause | Fix |
|---|---|---|
| 404 on `/api/chat` | `api/index.ts` missing from git | Ensure `api/index.ts` is committed and not gitignored |
| 500 on `/api/chat` | Build failed or env var missing | Check Vercel build logs; verify `OPENAI_API_KEY` is set |
| `ERR_MODULE_NOT_FOUND` | Server code not bundled | Ensure build script runs `bun build ... --target node` |
| Functions not detected | `api/lib/` accidentally committed | Keep `api/lib/` in `.gitignore`; only `api/index.ts` in git |
